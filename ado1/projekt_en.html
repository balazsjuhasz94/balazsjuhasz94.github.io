About the Project

Introduction

This is a hobby project that was essentially built over two weekends, using an entirely AI-written codebase — a practice nowadays known as "vibe coding".

The project's goal is to show the trends of 1% personal income tax donations made to civil organizations, and to give people a chance to browse and discover potential organizations to support.

The completed interactive visualizations are available at ado1stat.hu.

The categorization of organizations was performed automatically using an LLM, based on the organizations' stated goals available on civil.info.hu.

General Information

In 2025, a total of 20.3 billion HUF was donated to more than 37,000 organizations.

The donations are distributed very unevenly among organizations:

  • the top 100 organizations received 34% of the total (7.0 billion HUF)
  • the top 1,000 organizations received 59% (11.9 billion HUF)
  • the top 5,000 organizations received 79% (15.9 billion HUF)
  • the top 10,000 organizations received 88% (17.9 billion HUF)

I ultimately selected the 10,000 organizations with the most donations. This covers the vast majority of the total amount donated, and visualizing more data would not be very practical.

Project Steps

  1. Automated data collection (scraping) from civil.info.hu for the selected 10,000 organizations
  2. Categorization of organizations using a properly prompted LLM (large language model)
  3. Visualization of the collected data
  4. Creating a website about the project

An additional non-trivial part of the process was manually correcting occasionally miscategorized organizations — and even so, the result is not perfect. It is not always straightforward to assign an organization to a single category, as their profile may span multiple areas, and the categories themselves can overlap.

There was already some form of categorization available on civil.info.hu — I initially wanted to use it, but the categories were not well separated, and the categorization itself was not accurate at all.

The full source code of the project is available on GitHub.

Technical Details and Costs

I'll try to explain this as clearly as possible.

I essentially never looked at the generated code and accepted every change automatically. In most cases the AI succeeded on the first try, but even when it didn't, no request ever required more than three iterations, regardless of complexity.

I used Anthropic's Claude products throughout the project.

I subscribed to Claude Pro (€25.4/month), which gives access to the Claude Code AI assistant.

When used intensively without wanting to wait for rate limits, there are additional usage fees — I ended up spending an extra €48 in tokens.

Additionally, the categorization of organizations required calling Claude's model via API within a script. The cost is calculated based on the number of tokens used. I used approximately 12 million input tokens and 200,000 output tokens, which came to $12.29. In practice, this means I prepared (or rather, had prepared) a suitable prompt describing the model's task. You can find the prompt on the Prompts Used page.

The categorization worked quite well. The only category that seemed to need improvement was education, where the specific type of school often couldn't be determined from the goal description alone. There is also an option to let the model search the internet when uncertain. I used this slightly more expensive approach for the ~3,000 education-related foundations, which cost a total of $15.39.

Even after all this, some miscategorizations are possible; when this happened for a large organization, I corrected it manually. In total, I manually changed fewer than 10 categorizations.

For the AI assistant I used Sonnet 4.5, which is recommended for this type of work and is not the most expensive model. For categorization I used the cheapest Haiku model, which was reliably capable of assigning organizations to the predefined categories based on their goal descriptions and a suitable introductory prompt.

For visualization I used the Python plotly library, which I have used many times before for interactive visualizations. Interactivity is provided by the accompanying dash package.

For the website itself, I first had to register the domain name, which costs 2,527 HUF per year (for a .hu domain) through Tárhely.eu. Hosting is handled through a cloud provider, with the server costing €3.49/month. If more than ~100 people wanted to browse the site simultaneously, a larger server would be needed.

I had very little experience with web hosting, but it went smoothly with Claude Code's help.

  • NAV public data, 2025 — the list of civil organizations that received 1% personal income tax donations
  • civil.info.hu — Basic organization data (location, stated goals) and their share of donations in prior years
